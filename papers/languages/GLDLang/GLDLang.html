Abstract:

GLDLang (Graphical Logic Diagram Language) will be a graphical esoteric programming language, inspired by Piet. It will be in a concept map/tree diagram-like format, where different colored rectangular nodes will represent different parts of a program or statement. For example, a green square could represent the beginning of a program, a yellow square could represent an instruction, a red square could represent data or a value, where the data is stored as different colored pixels inside the square, etc. 1st children and their  parent nodes will be connected by a black line (0x000000). Sibling nodes, like one instruction to the next instruction, or the 1st parameter to the 2nd parameter, will be connected by a gray line. (0x888888)

Types of nodes:

  • Start of program
  • Instruction
  • Parameter

Types of parameters:

  • Reference
  • Array Reference
  • Data
  • Array Data
  • Expression
  • Variable
  • Array
  • User Input

Parameters:

Reference

Will be for referencing a non-array variable.

Will be depicted by a dark red rectangle with the variable address inside.

Color:         0x990000

Returns: The value stored in the referenced variable

Children: none.

Array Reference

Will be for referencing a value inside an array.

Will be depicted by a dark purple rectangle with the array address inside.

Color:         0x990099

Returns: The item in the array specified at the index given (the first child).

Children: Data, Reference, Array Reference, Expression

Data

Will be for storing data/values for variables.

Will be depicted by a bright red rectangle with the data, represented by a bunch of pixels, the color being the data, inside of it. Data will be read from left to right, top to down, like in English.

Color: 0xFF0000

Returns: The value stored inside of the red square.

Children: none

Array Data

Will be for storing data/values for arrays.

Will be depicted by a bright purple rectangle, with the children being the elements of the array.

Color: #FF00FF

Returns: An array of data specified by its children.

Children: Data, Reference, Expression, Array Reference, User Input

Expressions

Will be for calculating mathematical expressions with values and variables.

Depicted by a dark teal rectangle with the opcode / instruction type inside of it.

Color: 0x009999

Returns: The result of the expression.

Children: Data, Reference, Array Reference, Expression

User Input

Allows the user to input data, like “cin” in C. The return depends on what is entered.

Depicted by a bright blue rectangle with the type of data being entered inside.

Color: 0x0000FF

Returns: If the color inside is green (0x00FF00), it returns the string entered by the user. If the color inside is red (0xFF0000), it returns an integer entered by the user. (if the entered string cannot be converted into an integer, an error will be given.)

Children: none

Making acceptable and working code

Data types:

Data types will be inferred based on what instruction. For example, a print string instruction will infer that the parameter will be a string. A print char instruction will infer that the parameter will be a character. If the data is longer than just one character, it will throw an error. Most other instructions would infer an integer. Data will be stored like in normal computers, strings and characters in ascii format, positive integers in normal binary (first bit is always reserved for the sign of the integer (+ or -)) negative integers will be in the two's complement of the number. (https://www.geeksforgeeks.org/1s-2s-complement-binary-number/ for explanation) Integers will have a max size of 64 bits. Any integer above this size will throw an error.

Size of nodes:

For nodes that are just one solid color, like the start of a program, they can be resized however you want. For nodes that require an inside of just one solid color, that inside can be resized however you want, but the border must stay one pixel wide. For things like the data parameter that require multiple colors, they cannot be resized.

Other things related to the acceptability of a program:

  • Black (0x000000) and gray (0x888888) wires may be bent; they do not have to be straight. They only have to touch two nodes and two nodes only.
  • The wires can also have any thickness or any shape, as long as they only touch two nodes.
  • Nodes may not touch other nodes.
  • A node may only have two black wires and two gray wires maximum attached to it.
  • Nodes must always have a rectangular shape.

Examples of unacceptable code:

The red data node has a red border of 2 pixels. This would be acceptable, but it would assume that anything inside a 1 pixel border is part of the data, and would not produce the expected output.

There are three black wires attached to the print instruction, which also has a non-rectangular shape.

The print and halt instructions are touching, this is unacceptable.

Demonstrations

Hello World

Explanation:

The green rectangle represents the start of the program.

The yellow rectangles means an instruction, with whatever inside being the opcode.

The blue inside the yellow rectangle is an opcode that means it is printing something out.

The red rectangle represents a value or data, with whatever inside being the value, encoded in the colors of pixels, going from left to right, up to down, as in English.

“Hello world!” in ascii, converted to hexadecimal code, is 48 65 6c 6c 6f 20 77 6f 72 6c 64 21.

Changing the hexadecimal code to html hexadecimal colors, you get #48656C, #6C6F20, #776F72, and #6C6421, which are the colors of the 4 pixels inside the red rectangle.

The white-filled yellow rectangle stops the program. Without this, the program would run in a loop.

The black line between the start and the print instruction means that the print instruction is a child of the start, and that it is the first instruction. The black line between the print instruction and the data means that the data is a child of the print instruction, that it is the first parameter of the print instruction.

The grey line between the print and halt instructions mean that the halt instruction is the next child of the start, after the print instruction; The next instruction to execute after the print.

Truth Machine (https://esolangs.org/wiki/Truth-machine)

Explanation:

1) print "Enter a boolean (0/1):"

2) save the user's input as an integer as a new variable "a"

3) if 0 and variable "a" are equal, jump to instruction 7.

4) if 1 and variable "a" are equal, jump to instruction 9.

5) print "You did not enter a 0 or a 1."

6) stop the program.

7) print "0"

8) stop the program.

9) print "1"

10) jump to instruction 9.        

Basically, what this does is:

1) Ask the user to input a number, either a 0 or a 1.

2) If the user inputs 0, it prints “0” once.

3) If the user inputs 1, it prints “1” in an infinite loop.

4) If the user inputs neither a 0 or a 1, it prints “You did not enter a 0 or a 1.”
